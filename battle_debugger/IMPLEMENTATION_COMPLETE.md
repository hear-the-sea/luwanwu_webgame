# Battle Debugger Web Interface - Implementation Complete

## 概述

战斗调试器Web界面已全面完成，提供了强大的战斗模拟和分析工具。

## 已实现功能

### 1. 首页 (`/debugger/`)
- 显示所有预设配置卡片
- 提供三个主要入口：
  - 战斗模拟
  - 自定义配置
  - 参数调优

### 2. 战斗模拟 (`/debugger/simulate/`)
- 选择预设配置进行模拟
- 可调整战斗参数：
  - 屠戮倍率
  - 小兵攻击除数（对门客/对小兵）
  - 小兵防御除数
  - 五行相克倍率
  - 暴击率
- 支持指定随机种子
- 支持重复模拟（批量测试）

### 3. **自定义配置** (`/debugger/custom/`) ⭐ 新增
完全自定义战斗配置：

#### 门客配置
- 动态添加/删除门客
- 选择门客模板（按稀有度和职业分类）
- 设置等级（1-100）
- 自定义属性：
  - 武力
  - 智力
  - 防御
  - 敏捷
  - 幸运
- 点选技能（多选界面）

#### 护院配置
- 动态添加/删除护院
- 选择兵种类型
- 设置数量

#### 其他配置
- 双方科技等级
- 所有战斗可调参数
- 随机种子和重复次数

### 4. 战斗结果页面 (`/debugger/result/<id>/`)
#### 统计信息
- 总战斗次数
- 攻方/守方胜率
- 平均回合数
- 胜率可视化（进度条）

#### **详细战斗日志** ⭐ 增强
完全参照现有战报格式：
- 左右分栏布局（攻方/守方）
- 每个回合独立显示
- 事件编号和顺序
- 技能标签
- 战技标签（暴击、远程、狂狼必杀、治疗）
- 反击伤害显示（护身剑罡、反戈一击）
- 状态效果显示
- 伤害人数统计
- 响应式设计（移动端自适应）

#### 批量结果
- 表格显示所有战斗结果
- 每场战斗的胜者、回合数、兵损

### 5. 参数调优 (`/debugger/tune/`)
- 选择预设配置
- 选择要调优的参数
- 指定多个参数值
- 图表化展示不同参数值的影响

### 6. API接口
- `/debugger/api/guests/` - 获取所有门客模板
- `/debugger/api/skills/` - 获取所有技能
- `/debugger/api/troops/` - 获取所有护院类型

## 技术实现

### 后端
- **Django MVT架构**：标准Model-View-Template
- **配置系统**：YAML预设 + Python dataclass配置对象
- **缓存机制**：Django cache存储战斗结果（1小时过期）
- **URL路由**：RESTful风格URL设计

### 前端
- **Bootstrap 5**：响应式UI框架
- **FontAwesome**：图标库
- **原生JavaScript**：动态表单生成、API数据加载
- **CSS Grid**：战斗日志双栏布局

### 关键修复
1. **模型字段映射**：
   - 数据库模型使用 `base_attack`
   - 战斗系统使用 `force`
   - API层面进行映射转换

2. **导入优化**：
   - 配置类提升到模块级导入
   - 避免类型提示导致的循环导入问题

## 使用方法

### 启动服务器
```bash
python manage.py runserver
```

### 访问界面
```
http://127.0.0.1:8000/debugger/
```

### 快速测试
1. **使用预设配置**：
   - 点击首页的"战斗模拟"
   - 选择一个预设（如"balanced"）
   - 点击"开始模拟"
   - 查看详细战斗日志和统计数据

2. **自定义配置**：
   - 点击首页的"自定义配置"
   - 添加门客（选择模板、设置属性、配置技能）
   - 添加护院（选择兵种、设置数量）
   - 调整战斗参数
   - 点击"开始模拟"

3. **参数调优**：
   - 点击首页的"参数调优"
   - 选择预设配置
   - 选择要调优的参数
   - 输入多个参数值（逗号分隔）
   - 查看图表化对比结果

## 已知问题

1. **门客模板缺失警告**：
   - 某些预设配置引用的门客模板不存在于数据库
   - 不影响功能，会自动跳过缺失的门客

2. **WebSocket连接失败**：
   - 其他应用的WebSocket连接失败（notifications, online-stats）
   - 不影响战斗调试器功能
