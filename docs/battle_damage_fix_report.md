# 战斗系统问题修复报告

## 问题总结

用户报告了4个战斗系统问题：
1. 任务的门客属性是否正确生效
2. 门客攻击护院时存在伤害为0的情况
3. 护院攻击护院也存在伤害为0的情况（如1拳王攻击2000箭神）
4. 造成伤害超过剩余血量时只显示了剩余血量，要求显示真实伤害

## 问题诊断

### 问题1：门客属性生效 ✅ 正常

**测试结果**：
- 配置：等级50，属性加成10%
- 黑阶学士：攻击68 → 74 ✅
- 独孤求败：攻击334 → 367 ✅
- **结论**：门客等级和属性加成完全正常生效

### 问题2：门客攻击护院伤害为0 ✅ 正常

**测试结果**：
- 黑阶学士（攻击74）vs 300刀客
- 伤害：126，击杀：7
- **结论**：门客攻击护院完全正常

### 问题3：护院攻击护院伤害为0 ❌ BUG确认

**问题现象**：
- 1拳王（攻击11）vs 2000箭神
- 有效防御力：1200
- 防御减伤率：90.9%
- 实际伤害：2-3点（无法击杀）
- 显示伤害：**0**

**根本原因**：
1. **防御倍率过高**（battle/combat_math.py:122）
   - 旧公式：`multiplier = strength / 10.0`
   - 2000兵力 → 防御倍率200 → 有效防御1200
   - 导致1个单位完全无法穿透大规模部队的防御

2. **显示逻辑错误**（battle/simulation_core.py:379）
   - 计算：`display_damage = kills * per_unit_hp`
   - 当 kills=0 时，display_damage=0
   - 即使造成了2-3点伤害，也显示为0

### 问题4：伤害显示限制 ✅ 正常

**测试结果**：
- 独孤求败攻击100血的黑阶学士
- 显示伤害：622（真实伤害）
- **结论**：伤害显示正常，不存在限制为剩余血量的问题

## 修复方案

### 修复1：调整防御倍率公式

**文件**：`battle/combat_math.py:110-131`

**修改**：
```python
# 旧公式（线性增长）
multiplier = max(1.0, strength / 10.0)

# 新公式（平方根缩放）
multiplier = max(1.0, math.sqrt(strength) / 2.0)
```

**效果对比**：

| 兵力  | 旧倍率 | 旧防御(6) | 新倍率 | 新防御(6) | 减伤率变化 |
|-------|--------|-----------|--------|-----------|------------|
| 100   | 10     | 60        | 5      | 30        | 33% → 20%  |
| 1000  | 100    | 600       | 15.8   | 95        | 83% → 44%  |
| 2000  | 200    | 1200      | 22.4   | 134       | 91% → 53%  |
| 10000 | 1000   | 6000      | 50     | 300       | 98% → 71%  |

**设计理念**：
- 使用平方根缩放，避免线性增长导致的防御过高
- 保持数量优势，但不会出现完全无法伤害的情况
- 1000 vs 1000 势均力敌时，防御合理
- 1 vs 2000 悬殊时，仍有可见伤害

### 修复2：调整伤害显示逻辑

**文件**：`battle/simulation_core.py:377-383`

**修改**：
```python
# 旧逻辑
display_damage = int(kills * per_unit_hp)

# 新逻辑
if kills > 0:
    display_damage = int(kills * per_unit_hp)  # 显示等效伤害
else:
    display_damage = damage  # 无击杀时显示实际伤害
```

**效果**：
- 有击杀时：显示"击杀数 × 单位HP"（等效伤害）
- 无击杀时：显示实际造成的伤害（不再是0）
- 玩家可以清楚看到即使无法击杀，也在持续造成伤害
