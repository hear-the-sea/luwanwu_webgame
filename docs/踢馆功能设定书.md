## 一、功能概述

踢馆是玩家之间的PVP核心玩法，玩家可以在世界地图上查找其他玩家的庄园，进行侦察和出征（踢馆）操作。成功踢馆可以掠夺对方的部分资源和物品。

## 二、庄园位置系统

### 2.1 地区划分

玩家在创建角色/首次进入游戏时需要选择庄园所在地区，以中国行政区划为基础：

#### 直辖市（4个）
- 北京市、天津市、上海市、重庆市

#### 省份（23个）
- 河北省、山西省、辽宁省、吉林省、黑龙江省
- 江苏省、浙江省、安徽省、福建省、江西省
- 山东省、河南省、湖北省、湖南省、广东省
- 海南省、四川省、贵州省、云南省、陕西省
- 甘肃省、青海省、台湾省

#### 自治区（5个）
- 内蒙古自治区、广西壮族自治区、西藏自治区
- 宁夏回族自治区、新疆维吾尔自治区

#### 特别行政区（2个）【补充】
- 香港特别行政区、澳门特别行政区

#### 特殊地区（1个）【v1.5新增】
- 化外之地（供海外玩家选择）

### 2.2 坐标系统

- 每个地区拥有独立的坐标系统
- 坐标范围：X: 1-999, Y: 1-999
- 玩家注册时在所选地区内随机生成唯一坐标
- 坐标格式显示：`[地区名] (X, Y)`，例如：`[浙江省] (156, 423)`

### 2.3 坐标用途

- **距离计算**：用于计算出征/侦察所需时间
- **距离公式**：`distance = sqrt((x1-x2)^2 + (y1-y2)^2)`
- **跨区惩罚**【补充】：跨地区出征时，基础时间额外增加20%

### 2.4 庄园迁移【v1.5新增】

玩家可以通过消耗资源或道具迁移庄园到其他地区：

#### 迁移费用

根据庄园总资产等级收取不同的金条费用：

| 资产等级 | 总资产范围 | 金条消耗 |
|----------|-----------|---------|
| 匮乏 | < 10,000 | 1 |
| 一般 | 10,000 ~ 50,000 | 3 |
| 充裕 | 50,000 ~ 200,000 | 6 |
| 富足 | > 200,000 | 10 |

**总资产计算公式**（与侦察报告一致）：
```
总资产 = 粮食数量 + 银两数量 + 仓库物品总价值
仓库物品总价值 = Σ(物品数量 × 物品price)
```

#### 迁移规则

- 迁移后在新地区随机生成新坐标
- 迁移冷却时间：30天（防止频繁迁移）
- 迁移入口：设置界面
- 以下情况无法迁移：
  - 有出征中的队伍
  - 有敌军来袭
  - 处于新手保护期内
  - 迁移冷却中
- 迁移不影响庄园内的建筑、资源、门客等数据

## 三、地图查询系统

### 3.1 查询方式

#### 按庄园名称查询
- 支持模糊搜索
- 显示匹配的庄园列表（最多显示20条）【补充】

#### 按地区查询
- 选择直辖市/省份/自治区
- 显示该地区所有玩家庄园列表
- 支持分页浏览【补充】

#### 按坐标范围查询【补充】
- 输入中心坐标和搜索半径
- 显示范围内的所有庄园

### 3.2 庄园列表显示信息

| 字段 | 说明 |
|------|------|
| 庄园名称 | 玩家的庄园名称 |
| 地区 | 所在省份/直辖市 |
| 坐标 | (X, Y) |
| 声望等级 | 显示颜色标识（见3.3） |
| 距离 | 与自己庄园的距离 |

### 3.3 声望颜色系统

根据双方声望差值显示不同颜色。动态保护规则按**双方较低声望**所在区间判定：

| 双方较低声望区间 | 允许差值 | 颜色判定 |
|------|------|------|
| 0 ~ 4999 | ±1000 | 超出范围显示绿/红，范围内显示白 |
| 5000 ~ 9999 | ±2000 | 超出范围显示绿/红，范围内显示白 |
| 10000 ~ 14999 | ±3000 | 超出范围显示绿/红，范围内显示白 |
| 15000 ~ 19999 | ±4000 | 超出范围显示绿/红，范围内显示白 |
| 双方都 ≥ 20000 | 不受声望差保护 | 统一视为白色，可侦察/踢馆 |

颜色含义：

| 颜色 | 含义 | 可操作性 |
|------|------|----------|
| 🟢 绿色 | 对方相对过弱 | **不可侦察/踢馆** |
| ⚪ 白色 | 目标在可交互区间内 | 可侦察/踢馆 |
| 🔴 红色 | 对方相对过强 | **不可侦察/踢馆** |

### 3.4 声望系统（复用现有系统）

现有系统已实现声望机制，位于 `gameplay/models.py` 和 `gameplay/services/prestige.py`：

**现有字段**：
- `Manor.prestige` - 当前声望值（默认0）
- `Manor.prestige_silver_spent` - 累计花费银两（用于计算声望）

**现有计算规则**：
- 每累计花费 **1000两银两** 在建筑或技术上，增长 **1点声望**
- 公式：`prestige = prestige_silver_spent // 1000`

**踢馆功能扩展**【补充】：
```
踢馆相关声望变化（需要新增）：
- 踢馆胜利：+50 声望
- 踢馆失败：-30 声望
- 被踢馆胜利（防守成功）：+30 声望
- 被踢馆失败（防守失败）：-20 声望
```

## 四、侦察系统

### 4.1 侦察条件

- 消耗：1个探子（护院兵种 `scout`，从 `PlayerTroop` 中扣除）
- 目标限制：只能侦察白名玩家（按动态声望保护规则判定；双方声望都达到20000后不再受声望差保护）
- 冷却时间：对同一目标侦察间隔至少30分钟【补充】
- 新手保护：处于新手保护期内的玩家无法侦察他人，也无法被侦察

### 4.2 侦察成功率计算

```
基础成功率 = 50%
成功率修正 = (我方侦察术等级 - 对方侦察术等级) × 5%
最终成功率 = min(95%, max(10%, 基础成功率 + 成功率修正))
```

**侦察术等级**（复用现有技术系统 `PlayerTechnology`）：
- 技术key：`scout_art`（侦查术）
- 初始等级：0级
- 升级方式：消耗银两（通过现有技术升级系统）
- 最高等级：300级
- 每级提供：+10% 侦察/反侦察效果（`effect_per_level: 0.10`）

### 4.3 侦察时间

```
侦察时间（秒）= 距离 × 2 + 60
```

### 4.4 侦察结果

#### 侦察成功

通过消息系统发送侦察报告，包含：

| 信息类型 | 显示方式 | 说明 |
|----------|----------|------|
| 护院数量 | 模糊描述 | 少量(<100) / 旗鼓相当(100-500) / 较多(500-2000) / 庞大(>2000) |
| 门客数量 | 仅数量 | 例如："拥有门客 5 人" |
| 资源情况 | 模糊描述 | 匮乏/一般/充裕/富足（每种资源独立显示）|
| 建筑等级 | 不显示 | - |
| 门客详情 | 不显示 | - |

**资源模糊等级**【补充】：

侦察报告显示庄园总资产的模糊等级：

**总资产计算公式**：
```
总资产 = 粮食数量 + 银两数量 + 仓库物品总价值
仓库物品总价值 = Σ(物品数量 × 物品price)
```

**资产等级划分**：
- 匮乏：< 10,000
- 一般：10,000 ~ 50,000
- 充裕：50,000 ~ 200,000
- 富足：> 200,000

#### 侦察失败

- 我方：收到"侦察失败"消息，探子损失
- 对方：收到警告消息，内容包含：
  - 来源庄园名称
  - 来源地区和坐标
  - 侦察时间

## 五、踢馆（出征）系统

### 5.1 出征条件

- 目标限制：只能攻击白名玩家（按动态声望保护规则判定；双方声望都达到20000后不再受声望差保护）
- 门客要求：至少派出1名门客
- 门客状态：所有派出门客必须处于 `idle`（空闲）状态
- 护院要求：可选择携带护院数量（0~当前护院数）【补充】
- 出征上限：同时最多进行3次出征【补充】

### 5.2 出征时间计算

```
单程时间（秒）= 基础时间 + 距离 × 速度系数
基础时间 = 300秒（5分钟）
速度系数 = 3（每单位距离3秒）
跨区惩罚 = 同区×1.0 / 跨区×1.5

总出征时间 = 单程时间 × 2（往返）
```

**速度加成**【补充】：
- 门客平均敏捷影响速度：每10点敏捷减少1%时间
- 特定兵种加成：骑兵减少20%时间

### 5.3 战斗流程

1. **抵达目标**：单程时间结束后触发战斗
2. **战斗计算**：使用现有战斗系统（battle/services.py）
3. **结果判定**：根据战斗结果决定胜负
4. **返程**：战斗结束后自动返程

### 5.4 防守方配置【补充】

防守方自动使用：
- 所有 `idle`（空闲）状态的门客参战
- 当前所有护院参战（`PlayerTroop` 中的兵种）
- 防守阵型：默认阵型（后续可扩展自定义）

**门客状态枚举**（`GuestStatus`）：
- `idle`: 空闲（可出征/防守）
- `working`: 打工中（不可参战）
- `deployed`: 出征中（不可参战）
- `injured`: 重伤（不可参战）

### 5.5 战斗结果

#### 双方通用奖励（胜利方获得）

无论进攻方还是防守方，**胜利方**均可获得以下奖励：

| 奖励类型 | 说明 |
|----------|------|
| 经验果 | 根据战斗规模计算，每个可减少门客升级时间1分钟 |
| 装备回收 | 回收双方阵亡护院的部分装备 |

**经验果获得公式**：
```
经验果总数 = 护院经验果 + 门客经验果

护院经验果 = Σ(阵亡护院数量 × 护院基础生产时间 / 60) × 0.1
  - 计算双方所有阵亡护院（己方+敌方）
  - 护院基础生产时间参考 troop_templates.yaml 中的 base_duration（秒）
  - 示例：双方共100个刀客阵亡，base_duration=120秒，贡献 100×120/60×0.1 = 20 个经验果

门客经验果 = Σ(被击败门客等级 × 稀有度系数 × 战前血量占比) × 0.05
  - 计算双方所有被击败（HP归零）的门客（己方+敌方）
  - 战前血量占比 = 战斗开始时的HP / 门客最大HP
  - 稀有度系数（从低到高）：
    - black(黑): 0.5
    - gray(灰): 0.8
    - green(绿): 1.0
    - red(红): 1.5
    - blue(蓝): 2.0
    - purple(紫): 3.0
    - orange(橙): 5.0
  - 示例：双方共有1个50级蓝色门客被击败（战前血量80%），贡献 50×2.0×0.8×0.05 = 4 个经验果

注：经验果无上限，按实际战斗规模计算（双方损失越大，经验果越多）
```

**装备回收规则**：
```
回收概率 = 30%（每件装备独立判定）
回收来源 = 双方阵亡护院消耗的装备（参考 troop_templates.yaml 中的 recruit.equipment）
回收数量 = 阵亡护院数量 × 回收概率

示例：100个刀客阵亡，刀客需要 [equip_duandao, equip_buxie, equip_bupao]
  - equip_duandao 回收：100 × 30% = 30 个
  - equip_buxie 回收：100 × 30% = 30 个
  - equip_bupao 回收：100 × 30% = 30 个
```

#### 进攻方胜利（额外奖励）

| 奖励类型 | 说明 |
|----------|------|
| 资源掠夺 | 掠夺对方资源的10%~30%（随机）【补充】|
| 物品掠夺 | 概率掠夺对方仓库中 `tradeable=True` 的物品 |
| 声望变化 | 进攻方 +50，防守方 -20 |

**物品掠夺规则**【补充】：
```
掠夺概率 = 基础概率 × 物品稀有度系数
基础概率 = 20%
稀有度系数（对应 RARITY_ORDER，从低到高）：
  - black(黑): 1.0
  - gray(灰): 0.8
  - green(绿): 0.5
  - red(红): 0.35
  - blue(蓝): 0.2
  - purple(紫): 0.1
  - orange(橙): 0.05

每次踢馆最多掠夺3件物品
每件物品最多掠夺数量 = min(对方持有数量的20%, 5)
```

**资源掠夺上限**【补充】：
```
单次掠夺上限 = min(对方资源×30%, 我方仓库剩余容量, 10000)
```

| 经验果 | 按通用规则获得 |

#### 进攻方失败

| 物品损失 | 无 |

#### 防守方失败

| 结果 | 说明 |

## 六、撤退机制【补充】

### 6.1 撤退条件

- 只能在去程阶段撤退
- 抵达目标后无法撤退

### 6.2 撤退结果

- 立即开始返程
- 返程时间 = 已行进时间
- 不触发战斗
- 不损失资源/物品
- 声望变化：无

## 七、保护机制【补充】

### 7.1 新手保护

- 新玩家进入游戏后获得 **7天** 新手保护期
- 新手保护期间：
  - 无法被其他玩家侦察
  - 无法被其他玩家踢馆
  - 无法侦察其他玩家
  - 无法踢馆其他玩家
- 保护期到期后自动解除，无法提前解除

### 7.2 战败保护

- 被踢馆失败后获得30分钟保护时间
- 保护期间不会被再次踢馆
- 主动发起踢馆会解除保护

### 7.3 免战牌保护

玩家可使用「免战牌」道具主动开启保护：

| 道具 | 保护时长 | 说明 |
|------|---------|------|
| 免战牌·小 | 8小时 | 基础保护 |
| 免战牌·中 | 24小时 | 日常保护 |
| 免战牌·大 | 3天 | 长期保护 |

**免战牌规则**：
- 使用后立即生效，保护期间无法被侦察和踢馆
- 保护期间**无法**主动侦察或踢馆他人（双向限制）
- 保护期间可正常进行其他游戏活动（任务、建筑、招募等）
- 多个免战牌可叠加使用，时长累加
- 已在出征途中时无法使用免战牌
- 有敌军来袭时无法使用免战牌

### 8.1 新增模型

```python
# 庄园位置扩展（直接添加到 Manor 模型）
# 在 gameplay/models.py 的 Manor 模型中新增字段：
```

### 8.2 侦察消耗说明

侦察消耗的是护院兵种「探子」（`scout`），而非道具。

**现有探子兵种配置**（`data/troop_templates.yaml`）：
```yaml
- key: scout
  name: 探子
  description: 轻装侦察，行动迅捷，适合刺探敌情。
  base_attack: 1
  base_defense: 1
  base_hp: 10
  speed_bonus: 1
  priority: 0
  default_count: 50
```

**侦察时扣除逻辑**：
- 从 `PlayerTroop` 中查找 `troop_template.key = 'scout'` 的记录
- 扣除 `count` 数量 1
- 若探子数量不足则无法发起侦察
