### 关键公式：防御减伤（软上限公式）

**所有战斗类型统一使用软上限公式**：
```python
# 基础减伤
base = defense / (defense + 常数)

# 软上限：超过50%后收益减半
if base > 0.50:
    reduction = 0.50 + (base - 0.50) * 0.5
else:
    reduction = base

# 硬上限：最高75%
reduction = min(0.75, reduction)
```

| 战斗类型 | 防御常数 | 伤害倍率 | 说明 |

### 2.2 详细计算：100级紫色武门客

**门客属性（平衡配点，未计算装备加成）：**
- 武力439，智力100，敏捷80，防御313
- 攻击439，HP 18,050
- **注意：实际战斗中门客可穿戴装备，属性会更高**

#### 场景1：普攻模式（不放技能）

**vs 2000满科技箭神（单兵30攻/12防/90HP）：**

```
门客伤害 = 439（攻击）
对箭神减伤 = min(12 × 0.005, 0.75) = 0.06（单兵防御）
实际伤害 = 439 × (1 - 0.06) = 413
屠戮击杀 = 413 × 30（倍率） / 90（单位HP） = 138人/回合
```

#### 场景2：技能模式（奇策突袭）

**技能伤害计算：**
```
技能加成 = 220（base）+ (439×0.5 + 100×0.3 + 80×0.2) - (12×0.35 + 100×0.15)
        = 220 + 259.5 - 19.2
        = 460

总伤害 = 439（普攻）+ 460（技能）= 899
屠戮击杀 = 899 × 30 / 90 = 300人/回合
```

**技能触发率 = 0.8 + 50（运势）/ 1000 = 0.85**

**期望击杀 = 138 × 0.15 + 300 × 0.85 = 276人/回合**

**提升：+100%**

**附加效果：80%眩晕1个目标1回合**

#### 场景3：橙色门客AOE技能

**假设：橙色武门客（550攻击）使用"龙吟破阵"**

```
技能加成 ≈ 260 + (550×0.4 + 120×0.2 + 90×0.2 + 60×0.2) - (12×0.3 + 120×0.2)
        ≈ 260 + 282 - 27.6
        ≈ 514

总伤害 = 550 + 514 = 1064
对3个目标：
  - 打3组小兵：1064 × 30 / 90 × 3 = 1064人/回合
  - 打3个门客：每个造成1064伤害（秒杀低防门客）
```

## 五、平衡性问题总结

### 🔴 严重问题

#### 1. 防守方无限护院导致难度失控

**现象：**
- 任务/BOSS可以配置10门客+10,000护院（2.67倍兵力）
- 玩家即使满级15门客+满科技，仍然必输
- 护院数量差距压倒技能、装备、等级优势
- PVP补充：门客不会永久死亡，可反复派出耗掉对手护院；但无限护院让PVP变成长时间消耗战，胜负更多取决于玩家时间/粮耗而非单场对决

**影响：**
- 后期内容难度失衡
- 玩家培养意义降低（数值碾压一切）
- 策略深度降低（只看兵力对比）
- PVP体验被迫靠多次骚扰刷耐心，效率和乐趣都受影响

**建议修复：**
```python
# 方案A：防守方护院上限 = 门客数 × 300
defender_max_troops = len(defender_guests) × 300

# 方案B：护院数量差距递减机制
excess = defender_troops - attacker_troops
if excess > 1000:
    penalty = 1.0 - min(0.3, (excess - 1000) / 10000 × 0.3)
    defender_troops_effective = int(defender_troops * penalty)

# 方案C：在任务配置中明确限制
# data/mission_templates.yaml
defender_setup:
  max_troops_multiplier: 1.2  # 最多比玩家多20%
```

**推荐方案A**：简单直接，防守方300/门客 vs 玩家250/门客（+20%略优）
**PVP折中**：如要保留“多次骚扰消耗护院”的玩法，可仅对PVE/NPC强制上限；PVP采用软上限或补给冷却（如护院每小时最多恢复X%，超过门客数×300的部分衰减）

#### 2. 门客战斗贡献仍然偏低

**现象：**
- 即使计入技能，门客总输出占比仅10-15%
- 门客的核心价值仍是"槽位"而非"战斗力"
- 培养门客质量的激励不足

**影响：**
- 玩家只关心"门客数量"（升级悠嘻宝塔）
- 门客装备、技能、属性提升优先级低
- 养成系统价值降低

**建议修复：**
```python
# 方案A：门客光环系统
def apply_guest_auras(guests, troops):
    """每个门客为同侧护院提供+2%攻防"""
    if not guests:
        return
    multiplier = 1.0 + len(guests) * 0.02
    for troop in troops:
        troop.attack = int(troop.attack * multiplier)
        troop.defense = int(troop.defense * multiplier)

# 预期效果：
# 15门客提供：30%护院攻防加成
# 门客质量影响光环强度（高武力门客提供更强光环）

# 方案B：门客对小兵额外伤害
if actor.kind == "guest" and target.kind == "troop":
    attack_value = int(attack_value * 1.5)  # +50%

# 方案C：提高技能触发率和伤害
base_probability: 0.9  # 从0.8提升到0.9
```

**推荐方案A（光环系统）**：
- 门客数量提供基础光环
- 门客质量影响光环强度
- 增强配队策略深度

### 🟡 中等问题

**影响：**
- 门客无法发挥持续作用
- 技能释放次数有限
- 控制效果无法持续

**当前参数（已调整）：**
- 小兵打门客攻击倍率：兵力/2.5
- 小兵打门客防御公式：软上限公式
  - 基础：`defense / (defense + 200)`
  - 超过50%后收益减半
  - 硬上限75%
- 2000小兵击杀门客：
  - 无装备（313防御）：约1.7回合
  - 当前装备（488防御）：约2.2回合
  - 未来紫装（803防御）：约3.0回合

**建议修复：**
```python
# 方案A：HP系数提升（已讨论但未实施）
max_hp = base_hp + defense × 100  # 从×50改为×100

# 方案B：门客减伤被动
if target.kind == "guest" and damage > target.max_hp * 0.1:
    damage = int(damage * 0.85)  # 大伤害减免15%

# 方案C：门客专属格挡机制
if target.kind == "guest" and rng.random() < 0.15:
    damage = int(damage * 0.5)  # 15%几率格挡50%伤害
```

**可选实施**：取决于测试反馈

### 优先级1：限制防守方护院数量（关键）

**目标：平衡进攻/防守方战力，让战斗有策略性**

```python
# battle/services.py 新增函数
def cap_defender_troops(defender_guests: int, defender_loadout: Dict[str, int]) -> Dict[str, int]:
    """
    限制防守方护院总数为门客数 × 300

    Args:
        defender_guests: 防守方门客数量
        defender_loadout: 防守方护院配置

    Returns:
        裁剪后的护院配置
    """
    max_capacity = defender_guests * 300
    total_troops = sum(defender_loadout.values())

    if total_troops <= max_capacity:
        return defender_loadout

    # 按比例缩减
    scale = max_capacity / total_troops
    capped = {}
    for key, count in defender_loadout.items():
        capped[key] = max(1, int(count * scale))

    return capped
```

```python
# battle/combatants.py 新增函数
def apply_guest_auras(guests: List[Combatant], troops: List[Combatant]) -> None:
    """
    门客为同侧护院提供光环加成

    基础光环：每个门客提供+2%攻防
    质量加成：高武力/智力门客提供额外加成

    示例：
    - 15个门客：基础+30%攻防
    - 15个高武力门客（平均400+）：额外+5%
    - 总计：+35%攻防
    """
    if not guests or not troops:
        return

    # 基础光环：每个门客+2%
    base_multiplier = 1.0 + len(guests) * 0.02

    # 质量加成：根据门客平均武力/智力
    avg_force = sum(g.force_attr for g in guests) / len(guests)
    avg_intellect = sum(g.intellect_attr for g in guests) / len(guests)
    quality_bonus = (avg_force + avg_intellect) / 10000  # 平均500属性 = +5%

    total_multiplier = base_multiplier + quality_bonus

    for troop in troops:
        troop.attack = int(troop.attack * total_multiplier)
        troop.defense = int(troop.defense * total_multiplier)
```
