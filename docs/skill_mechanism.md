# 门客技能生效机制详解

## 技能类型

技能分为两种类型：

### 1. 主动技能（Active）
- **特点**：每回合最多触发1个
- **触发方式**：运势决定触发率
- **效果**：直接造成伤害加成，可能附加状态效果（眩晕、冻结等）
- **示例**：
  - 龙吟破阵（dragon_roar）：威力260，对3个敌人
  - 奇策突袭（stratagem_burst）：威力220，对1个敌人，可眩晕

### 2. 被动技能（Passive）
- **特点**：可以同时触发多个
- **触发方式**：运势决定触发率
- **效果**：属性加成、特殊效果
- **示例**：
  - 防御型被动
  - 回血型被动
  - 减伤型被动

## 触发概率计算（核心公式）

**公式**：
```
触发率 = min(0.95 × (运势 / 300) ^ 0.85, 0.95)
```

**关键设计**：
- 技能本身**不再有独立触发率**，统一由门客运势决定
- 运势是**先天属性**，仅通过模板基础值和部分装备获得
- 幂函数设计：低运势段差距大，高运势段收敛到95%上限
- 运势300即可达到95%触发率上限

**触发率速查表**：

| 运势 | 触发率 | 典型角色 |
|------|--------|----------|
| 50   | 20.2%  | 低运武将 |
| 65   | 25.3%  | 普通武将 |
| 80   | 30.1%  | 中等武将 |
| 100  | 36.2%  | 高运武将 |
| 130  | 45.1%  | 低运文官 |
| 150  | 52.6%  | 普通文官 |
| 180  | 60.8%  | 高运文官 |
| 200  | 66.9%  | 极高运势 |
| 300+ | 95.0%  | 上限     |

**设计理念**：
- 武将依赖普攻和兵力优势，技能为辅助（约20-36%概率触发）
- 文官通过高运势获得频繁技能释放，弥补武力不足（约45-61%概率触发）
- 文官触发率约为武将的**3倍**，体现职业差异

## 多技能触发规则

位置：`battle/skills.py:14-48`

```python
def trigger_skills(attacker, rng: random.Random) -> List[Dict]:
    """
    技能触发规则：
    1. 根据运势计算统一触发率
    2. 遍历门客的所有技能，独立判定
    3. 被动技能：所有触发成功的都会生效
    4. 主动技能：只随机选择1个生效
    5. 返回触发的技能列表（被动 + 1个主动）
    """
    triggered: List[Dict] = []
    triggered_active: List[Dict] = []

    # 运势决定触发率，幂函数公式
    # 低运势差距大，高运势收敛到95%上限
    luck = getattr(attacker, "luck", 50)
    chance = min(0.95, 0.95 * pow(luck / 300, 0.85))

    for skill in attacker.skills:
        if rng.random() > chance:
            continue  # 未触发

        if _is_active_skill(skill):
            triggered_active.append(skill)  # 主动技能暂存
        else:
            triggered.append(skill)  # 被动技能直接加入

    # 主动技能只选1个
    if triggered_active:
        triggered.append(rng.choice(triggered_active))

    return triggered
```

## 具体场景示例

### 场景1：武将有2个主动技能
**配置**：
```yaml
guest:
  archetype: military
  luck: 65  # 武将典型运势
skills:
  - dragon_roar      # 主动，对3人
  - stratagem_burst  # 主动，对1人眩晕
```

**运势65的武将，每回合触发流程**：

1. **计算触发率**：
   - chance = 0.95 × (65/300)^0.85 = 25.3%

2. **第一次判定**（龙吟破阵）：
   - 随机数 0.00-1.00
   - 如果 < 0.253 → 触发成功，加入候选

3. **第二次判定**（奇策突袭）：
   - 随机数 0.00-1.00
   - 如果 < 0.253 → 触发成功，加入候选

4. **最终选择**：
   - 如果两个都触发：随机选1个释放
   - 如果只触发1个：释放该技能
   - 如果都未触发：普通攻击

**结果分析**：
- 单个技能触发概率：25.3%
- 至少触发1个技能的概率：1 - (1-0.253)² = 44.2%
- 大约每2-3回合释放1次技能

### 场景2：文官有2个主动技能
**配置**：
```yaml
guest:
  archetype: civil
  luck: 150  # 文官典型运势
skills:
  - dragon_roar      # 主动，对3人
  - stratagem_burst  # 主动，对1人眩晕
```

**运势150的文官，每回合触发流程**：

1. **计算触发率**：
   - chance = 0.95 × (150/300)^0.85 = 52.6%

2. **技能判定**：
   - 每个技能独立判定，52.6%概率触发

**结果分析**：
- 单个技能触发概率：52.6%
- 至少触发1个技能的概率：1 - (1-0.526)² = 77.5%
- 大约每1-2回合释放1次技能
- **对比武将**：文官触发技能频率约为武将的1.75倍

### 场景3：门客有1个主动 + 1个被动
**配置**：
```yaml
skills:
  - dragon_roar        # 主动
  - defensive_stance   # 被动，减伤20%
```

**触发流程**：

1. **主动技能判定**（龙吟破阵）：
   - 根据运势判定触发
   - 触发成功 → 对3人造成伤害

2. **被动技能判定**（防御姿态）：
   - 独立判定（同样的触发率）
   - 触发成功 → 同时生效减伤

**结果**：
- 主动和被动可以同时生效
- 最理想情况：龙吟破阵打出伤害 + 防御姿态减伤

## 文武门客强度对比

### 武将（Military）特点：
- **运势范围**：约50-100
- **技能触发率**：20.2%-36.2%
- **优势**：
  - 武力高，普攻伤害大
  - 带兵能力强，更多护院
  - 依靠稳定的物理输出
- **技能定位**：锦上添花，偶尔爆发

### 文官（Civil）特点：
- **运势范围**：约130-180
- **技能触发率**：45.1%-60.8%
- **优势**：
  - 智力高，技能伤害大
  - 频繁触发技能
  - 控制技能（眩晕、冻结）更常见
- **技能定位**：核心输出手段

### 平衡分析：
- 文官技能触发率约为武将的 **2-3倍**
- 武将有更高的基础攻击力作为补偿
- 两种职业走不同的战斗路线

## 技能配置建议

### 推荐配置1：双主动（平衡型）
```yaml
skills:
  - dragon_roar      # 群攻（对3人）
  - stratagem_burst  # 单体控制（眩晕）
```
**分析**：
- 武将（25.3%触发率）：至少释放1个技能的概率约44%
- 文官（52.6%触发率）：至少释放1个技能的概率约78%
- 自动选择群攻或控制，灵活应对

### 推荐配置2：主动+被动（稳定型）
```yaml
skills:
  - dragon_roar      # 主动输出
  - defensive_aura   # 被动减伤/回血
```
**优势**：
- 主动技能保证输出
- 被动技能提供持续效果
- 攻守兼备

### 不推荐配置：3个以上主动技能
```yaml
skills:
  - skill_1  # 主动
  - skill_2  # 主动
  - skill_3  # 主动
```
**问题**：
- 每回合依然只能释放1个
- 浪费技能槽位
- 不如配置被动技能获得持续收益

## 运势获取途径

运势是先天属性，获取途径有限：

1. **模板基础值**：门客模板的 `base_luck` 决定初始运势
2. **装备加成**：部分装备提供运势加成
3. **套装效果**：特定套装可能提供运势加成

**注意**：运势**不会随等级成长**，这是与其他属性的关键区别。

## 总结

**核心原则**：
1. 触发率由运势决定：`0.95 × (luck/300)^0.85`，上限95%
2. 技能本身不再有独立触发率
3. 被动技能全部生效（如果触发）
4. 主动技能每回合只选1个（如果触发）

**职业差异**：
- 武将：低运势（20-36%触发），依赖普攻
- 文官：高运势（45-61%触发），依赖技能

**配置建议**：
- 2个主动技能 = 最佳平衡
- 1主动 + N被动 = 稳定输出
- 3+主动技能 = 浪费槽位
